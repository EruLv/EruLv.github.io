9.static

当一个事物声明是static时，意味着这个域或方法不会与包含它的那个对象关联在一起。即无论创建多少个包含它的对象，系统只给它分配一个存储空间。

class test{

}

class Main {
    public static void main(String args[]){
        test t = new test();
        t.a ++;
        test tt = new test();
        tt.a ++;
        test.a ++;    //可以直接通过类名调用静态方法
        System.out.println(t.a);
        System.out.println(tt.a);
    }
}

2.访问权限修饰词

2.1包访问权限(默认权限)

没有访问权限修饰词，则默认为包访问权限(friendly),则当前包中的其他类堆那个成员对具有访问权限。对于包外的所有类都是不可见的。

如果你没有设置当前文件所属的包，即没有定义package xxxx,则位于同一目录的这样文件被java看作位于隶属于该目录的默认包中，于是对于同级目录下的没有设置包的文件来说，它们有包访问权限。

2.2 public

在任何位置都可以访问。

2.3 private

除了包含该成员的类之外，其他任何类都无法访问这个成员。

2.4 protected

继承的子类可以访问基类中的protected的成员（当然也可以访问public的）。

protected也提供包继承，即同一个包内的类也可以访问含有protected声明的成员。

3.注意，类(外部类)不可以时protected或者privated.可以把构造器设置为private,阻止其他类创建该类的实例。

8.final关键字

1.数据

final代表常量，如果修饰基本类型，则表示该数据不能修改，如果修饰对象，表示改引用不能指向其他对象(但对象可以改变)。

final static命名一般是大写字幕+下划线。

final必须在定义时初始化。当然，如果作为类成员变量，可以先不初始化，但一定要在构造函数里初始化。

class Test{

    private final int i;
    Test(){
        i = 1;
    }
}

2.方法

final修饰方法作用和private关键字类似，可以防止子类覆盖。

class WithFinals {
    // Identical to "private" alone:
    private final void f() { print("WithFinals.f()"); }
    // Also automatically "final":
    public final void g() { print("WithFinals.g()"); }
}

class OverridingPrivate extends WithFinals {
    private final void f() {
        print("OverridingPrivate.f()");
    }
    public void g() {   //会报错
        print("OverridingPrivate.g()");
    }
}

3.类

2.“覆盖”私有方法

public class PrivateOverride {
  private void f() { print("private f()"); }    
  public static void main(String[] args) {
    PrivateOverride po = new Derived();
    po.f();
  }
}

class Derived extends PrivateOverride {
  public void f() { print("public f()"); }    
} /* Output:

上面的基类中f()是私有的，与final等价，下面继承的方法看作是个全新的方法，基类中的方法f()在导出类中不可见，不能被重载，只有非private方法才可以被覆盖。

满足多态的条件:

  • 基类和子类中都有非private方法f().
  • A a = new B()形式，其中B继承A。

另外，向上转型后不能访问子类中存在但基类中不存在的方法。

3.“域”(类变量)访问

如果你直接访问某个域，这个访问就会在编译期间进行解析，不具有多态性。

class Super {
  public int field = 0;
  public int getField() { return field; }
}

class Sub extends Super {
  public int field = 1;
  public int getField() { return field; }
  public int getSuperField() { return super.field; }
}

public class FieldAccess {
  public static void main(String[] args) {
    Super sup = new Sub(); // Upcast
    System.out.println("sup.field = " + sup.field +
      ", sup.getField() = " + sup.getField());
    Sub sub = new Sub();
    System.out.println("sub.field = " +
      sub.field + ", sub.getField() = " +
      sub.getField() +
      ", sub.getSuperField() = " +
      sub.getSuperField());
  }
}
/* Output:
sup.field = 0, sup.getField() = 1
sub.field = 1, sub.getField() = 1, sub.getSuperField() = 0
*///:~

向上转型后，访问sup的域，因为sup是Super类，所以访问的是Super的类变量。

构造器是static的，因此不具有多态性。

构造器调用顺序:

从上往下初始化基类构造器->初始化当前类的变量->初始化当前类的构造器。

5.清理

如果需要显示地写辣鸡回收方法，子类覆盖父类的请理方法，子类的清理方法中一定要添加super().请理()方法，否则父类的清理方法一直不会执行。

6.构造器中调用被覆盖的类

class Glyph {
  void draw() { print("Glyph.draw()"); }
  Glyph() {
    print("Glyph() before draw()");
    draw();
    print("Glyph() after draw()");
  }
}  

class RoundGlyph extends Glyph {
  private int radius = 1;
  RoundGlyph(int r) {
    radius = r;
    print("RoundGlyph.RoundGlyph(), radius = " + radius);
  }
  void draw() {
    print("RoundGlyph.draw(), radius = " + radius);
  }
}  

public class PolyConstructors {
  public static void main(String[] args) {
    new RoundGlyph(5);
  }
} /* Output:
Glyph() before draw()
RoundGlyph.draw(), radius = 0
Glyph() after draw()
RoundGlyph.RoundGlyph(), radius = 5
*///:~

7.协变返回类型

class Grain {
  public String toString() { return "Grain"; }
}

class Wheat extends Grain {
  public String toString() { return "Wheat"; }
}

class Mill {
  Grain process() { return new Grain(); }
}

class WheatMill extends Mill {
  Wheat process() { return new Wheat(); }
}

public class CovariantReturn {
  public static void main(String[] args) {
    Mill m = new Mill();
    Grain g = m.process();
    System.out.println(g);
    m = new WheatMill();
    g = m.process();
    System.out.println(g);
  }
} /* Output:
Grain
Wheat
*///:~

8.向下转型

class Useful {
  public void f() {}
  public void g() {}
}

class MoreUseful extends Useful {
  public void f() {}
  public void g() {}
  public void u() {}
  public void v() {}
  public void w() {}
}  

public class RTTI {
  public static void main(String[] args) {
    Useful[] x = {
      new Useful(),
      new MoreUseful()
    };
    x[0].f();
    x[1].g();
    // Compile time: method not found in Useful:
    x[1].u();    //编译报错
    ((MoreUseful)x[1]).u(); //通过
    ((MoreUseful)x[0]).u(); // 抛出异常
  }
} ///:~
